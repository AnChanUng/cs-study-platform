## 질문
데이터베이스 정규화란 무엇이며, 각 정규형에 대해 설명해주세요.

## 핵심 키워드
- 함수 종속성
- 제1정규형 (1NF)
- 제2정규형 (2NF)
- 제3정규형 (3NF)
- BCNF
- 반정규화

## 답변
정규화(Normalization)란 관계형 데이터베이스에서 데이터의 중복을 최소화하고 무결성을 보장하기 위해 테이블을 분해하는 과정입니다. 이상 현상(Anomaly)을 방지하기 위해 수행합니다.

### 함수 종속성 (Functional Dependency)
정규화의 기반이 되는 개념으로, 속성 X의 값이 속성 Y의 값을 유일하게 결정할 때 "Y는 X에 함수적으로 종속된다"고 합니다 (X → Y).

### 정규형의 종류

**제1정규형 (1NF)**
모든 속성의 도메인이 원자값(Atomic Value)만을 가져야 합니다. 하나의 컬럼에 여러 값을 넣으면 안 됩니다.

```
// 위반 예시
| 학생 | 수강과목        |
|------|----------------|
| 홍길동 | 수학, 영어, 과학 |

// 1NF 만족
| 학생 | 수강과목 |
|------|---------|
| 홍길동 | 수학    |
| 홍길동 | 영어    |
| 홍길동 | 과학    |
```

**제2정규형 (2NF)**
1NF를 만족하고, 부분 함수 종속을 제거합니다. 기본키의 일부분에만 종속되는 속성이 없어야 합니다.

```
// 위반 예시 (학번, 과목코드가 복합키)
| 학번 | 과목코드 | 성적 | 학과 |
// 학과는 학번에만 종속 → 부분 함수 종속

// 2NF 만족: 테이블 분리
학생(학번, 학과)
수강(학번, 과목코드, 성적)
```

**제3정규형 (3NF)**
2NF를 만족하고, 이행적 함수 종속을 제거합니다. A → B → C일 때, A → C인 이행적 종속이 없어야 합니다.

```
// 위반 예시
| 학번 | 학과 | 학과사무실 |
// 학번 → 학과 → 학과사무실 (이행적 종속)

// 3NF 만족
학생(학번, 학과)
학과(학과, 학과사무실)
```

**BCNF (Boyce-Codd Normal Form)**
3NF를 만족하고, 모든 결정자가 후보키여야 합니다. 3NF보다 더 엄격한 조건입니다.

### 반정규화 (Denormalization)
정규화가 항상 최선은 아닙니다. 과도한 정규화는 JOIN 연산을 증가시켜 성능을 저하시킬 수 있습니다. 읽기 성능이 중요한 경우 의도적으로 중복을 허용하는 반정규화를 수행하기도 합니다.

### 면접 팁
정규화의 목적(이상 현상 방지)과 트레이드오프(성능 저하)를 함께 설명하세요. 실무에서는 보통 3NF까지 적용하고, 성능이 필요한 부분에서 선택적으로 반정규화를 수행합니다.

## 꼬리 질문

2. 정규화와 성능의 트레이드오프를 실무에서 어떻게 해결하나요?
3. BCNF와 3NF의 차이는 무엇인가요?

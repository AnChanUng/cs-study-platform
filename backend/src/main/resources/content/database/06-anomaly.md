## 질문
데이터베이스에서 이상(Anomaly)현상이란 무엇이며, 어떤 종류가 있나요?

## 핵심 키워드
- 삽입 이상 (Insertion Anomaly)
- 갱신 이상 (Update Anomaly)
- 삭제 이상 (Deletion Anomaly)
- 데이터 중복
- 정규화

## 답변
이상(Anomaly)이란 데이터베이스 테이블의 설계가 잘못되어 데이터를 삽입, 갱신, 삭제할 때 발생하는 논리적 오류를 말합니다. 주로 정규화가 충분히 이루어지지 않은 테이블에서 발생합니다.

### 예시 테이블

| 학번 | 이름 | 학과 | 학과전화 | 과목코드 | 성적 |
|------|------|------|---------|---------|------|
| 100 | 홍길동 | 컴공 | 1234 | CS101 | A |
| 100 | 홍길동 | 컴공 | 1234 | CS102 | B |
| 200 | 김영희 | 전자 | 5678 | EE201 | A |

### 이상의 종류

**1. 삽입 이상 (Insertion Anomaly)**
새로운 데이터를 삽입할 때 불필요한 데이터도 함께 삽입해야 하는 문제입니다.

예를 들어, 새로운 학과를 등록하고 싶은데 아직 해당 학과에 속한 학생이 없으면 학번, 이름, 과목코드, 성적 등에 NULL을 넣어야 합니다. 기본키에 NULL이 들어갈 수 없다면 삽입 자체가 불가능합니다.

**2. 갱신 이상 (Update Anomaly)**
중복된 데이터 중 일부만 수정되어 데이터 불일치가 발생하는 문제입니다.

홍길동의 학과 전화번호가 변경되면 홍길동과 관련된 모든 행(2개)을 수정해야 합니다. 하나만 수정하면 같은 학과인데 전화번호가 다른 모순이 발생합니다.

```sql
-- 하나만 수정하면 불일치 발생
UPDATE student_course SET 학과전화 = '9999'
WHERE 학번 = 100 AND 과목코드 = 'CS101';
-- CS102 행은 여전히 1234
```

**3. 삭제 이상 (Deletion Anomaly)**
데이터를 삭제할 때 의도하지 않은 다른 데이터도 함께 삭제되는 문제입니다.

김영희가 EE201 수강을 취소하면 해당 행을 삭제해야 하는데, 이때 김영희의 학생 정보(이름, 학과)와 전자학과의 전화번호 정보까지 모두 사라집니다.

### 해결 방법

**정규화를 통한 테이블 분리:**

```sql
-- 학생 테이블
CREATE TABLE student (
    학번 INT PRIMARY KEY,
    이름 VARCHAR(50),
    학과코드 VARCHAR(10) REFERENCES department(학과코드)
);

-- 학과 테이블
CREATE TABLE department (
    학과코드 VARCHAR(10) PRIMARY KEY,
    학과명 VARCHAR(50),
    학과전화 VARCHAR(20)
);

-- 수강 테이블
CREATE TABLE enrollment (
    학번 INT REFERENCES student(학번),
    과목코드 VARCHAR(10),
    성적 CHAR(1),
    PRIMARY KEY (학번, 과목코드)
);
```

이렇게 분리하면 각 정보가 한 곳에만 저장되어 이상 현상이 해결됩니다.

### 면접 팁
이상 현상은 정규화의 필요성을 설명할 때 핵심적인 근거가 됩니다. 세 가지 이상 현상을 구체적인 예시와 함께 설명하고, 정규화로 어떻게 해결하는지 연결하여 답변하세요.

## 꼬리 질문

3. 반정규화를 하면서도 이상 현상을 최소화하는 방법은 무엇인가요?

## 질문
운영체제(OS)란 무엇이며, 주요 역할에 대해 설명해주세요.

## 핵심 키워드
- 시스템 자원 관리
- 커널 (Kernel)
- 사용자 인터페이스
- 프로세스 관리
- 메모리 관리

## 답변
운영체제(Operating System)는 컴퓨터 하드웨어와 사용자(응용 프로그램) 사이에서 중재자 역할을 하는 시스템 소프트웨어입니다. 컴퓨터 시스템의 자원을 효율적으로 관리하고, 사용자에게 편리한 인터페이스를 제공합니다.

### 운영체제의 주요 역할

**1. 프로세스 관리**
- 프로세스의 생성, 실행, 종료를 관리합니다
- CPU 스케줄링을 통해 여러 프로세스에 CPU 시간을 할당합니다
- 프로세스 간 동기화와 통신(IPC)을 지원합니다

**2. 메모리 관리**
- 프로세스에 메모리 공간을 할당하고 회수합니다
- 가상 메모리를 통해 물리 메모리보다 큰 주소 공간을 제공합니다
- 페이징, 세그먼테이션 기법으로 메모리를 관리합니다

**3. 파일 시스템 관리**
- 파일과 디렉토리의 생성, 삭제, 접근을 관리합니다
- 디스크 공간을 할당하고 관리합니다
- 파일 접근 권한을 제어합니다

**4. 입출력(I/O) 관리**
- 다양한 입출력 장치를 제어하고 관리합니다
- 디바이스 드라이버를 통해 하드웨어 추상화를 제공합니다
- 버퍼링, 캐싱, 스풀링으로 I/O 효율을 높입니다

**5. 보안 및 보호**
- 사용자 인증과 접근 제어를 수행합니다
- 프로세스 간 메모리 보호를 제공합니다

### 커널 (Kernel)
운영체제의 핵심 부분으로, 항상 메모리에 상주하며 하드웨어와 직접 상호작용합니다.

```
[사용자 응용 프로그램]
        ↕ (시스템 콜)
    [커널(Kernel)]
        ↕
    [하드웨어]
```

**커널의 종류:**
- **모놀리식 커널**: 모든 서비스가 커널 공간에서 실행 (Linux)
- **마이크로 커널**: 최소한의 기능만 커널에, 나머지는 사용자 공간에서 실행 (Mach)
- **하이브리드 커널**: 두 방식의 혼합 (Windows NT)

### 사용자 모드 vs 커널 모드
- **사용자 모드(User Mode)**: 제한된 명령만 실행 가능
- **커널 모드(Kernel Mode)**: 모든 하드웨어 접근이 가능
- 시스템 콜을 통해 사용자 모드에서 커널 모드로 전환됩니다

### 면접 팁
운영체제의 목적을 "자원 관리의 효율성"과 "사용자 편의성" 두 가지 관점에서 설명하세요. 커널 모드와 사용자 모드의 분리가 왜 필요한지(보안, 안정성)도 설명할 수 있으면 좋습니다.

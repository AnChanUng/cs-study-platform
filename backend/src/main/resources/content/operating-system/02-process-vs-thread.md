## 질문
프로세스와 스레드의 차이점에 대해 설명해주세요.

## 핵심 키워드
- 프로세스: 독립적 메모리 공간
- 스레드: 공유 메모리 공간
- 멀티프로세스 vs 멀티스레드
- 컨텍스트 스위칭
- 동기화 문제

## 답변

### 프로세스 (Process)
프로세스는 실행 중인 프로그램으로, 운영체제로부터 자원을 할당받는 작업의 단위입니다.

**특징:**
- 독립적인 메모리 공간을 가짐 (Code, Data, Stack, Heap)
- 프로세스 간에는 메모리를 공유하지 않음
- 프로세스 간 통신에는 IPC(Inter-Process Communication)가 필요
- 하나의 프로세스가 다른 프로세스에 영향을 주지 않음

### 스레드 (Thread)
스레드는 프로세스 내에서 실행되는 흐름의 단위입니다.

**특징:**
- 프로세스 내에서 Code, Data, Heap 영역을 공유
- 각 스레드는 독립적인 Stack과 PC(Program Counter)를 가짐
- 스레드 간 자원 공유로 통신이 간단
- 하나의 스레드에 문제가 생기면 같은 프로세스의 다른 스레드에도 영향

```
프로세스 메모리 구조:
┌─────────────────────────┐
│       Code 영역         │  ← 스레드 간 공유
├─────────────────────────┤
│       Data 영역         │  ← 스레드 간 공유
├─────────────────────────┤
│       Heap 영역         │  ← 스레드 간 공유
├──────┬──────┬───────────┤
│Stack1│Stack2│ Stack3    │  ← 스레드별 독립
└──────┴──────┴───────────┘
 Thread1 Thread2 Thread3
```

### 비교

| 구분 | 프로세스 | 스레드 |
|------|---------|--------|
| 메모리 | 독립 공간 | 공유 (Stack만 독립) |
| 통신 | IPC 필요 | 공유 메모리로 간단 |
| 생성 비용 | 높음 | 낮음 |
| 컨텍스트 스위칭 | 비용 높음 | 비용 낮음 |
| 안정성 | 하나가 죽어도 다른 프로세스 영향 없음 | 하나가 죽으면 전체 영향 |

### 멀티프로세스 vs 멀티스레드

**멀티프로세스:**
- 안정성이 높음 (프로세스 간 독립)
- 메모리 사용량이 큼
- 컨텍스트 스위칭 비용이 큼
- 예: Chrome 브라우저 (탭마다 별도 프로세스)

**멀티스레드:**
- 자원 공유로 효율적
- 컨텍스트 스위칭 비용이 적음
- 동기화 문제에 주의 필요 (Race Condition, Deadlock)
- 예: 웹 서버의 요청 처리

```java
// 멀티스레드 예시
class MyThread extends Thread {
    public void run() {
        System.out.println("Thread: " + Thread.currentThread().getName());
    }
}

// 메인에서 스레드 생성
MyThread t1 = new MyThread();
MyThread t2 = new MyThread();
t1.start();
t2.start();
```

### 면접 팁
"프로세스는 자원 할당의 단위, 스레드는 CPU 스케줄링의 단위"라는 핵심을 기억하세요. 스레드가 Stack만 독립적으로 가지는 이유(각 스레드가 독립적인 실행 흐름을 가지므로)도 설명할 수 있으면 좋습니다.

## 질문
시스템 콜이란 무엇이며, 어떻게 동작하나요?

## 핵심 키워드
- 사용자 모드 → 커널 모드 전환
- 트랩 (Trap)
- 시스템 콜 인터페이스
- 프로세스/파일/장치 관련 시스템 콜
- fork(), exec(), open(), read()

## 답변
시스템 콜(System Call)은 운영체제가 제공하는 서비스에 접근하기 위해 사용자 프로그램이 커널에 요청하는 인터페이스입니다. 사용자 모드에서 직접 접근할 수 없는 하드웨어 자원이나 커널 기능을 사용할 때 필요합니다.

### 동작 과정

```
1. 사용자 프로그램에서 시스템 콜 호출
2. 소프트웨어 인터럽트(Trap) 발생
3. 사용자 모드 → 커널 모드 전환
4. 시스템 콜 번호를 기반으로 해당 핸들러 실행
5. 커널이 요청된 작업 수행
6. 결과를 반환하고 커널 모드 → 사용자 모드 전환
7. 사용자 프로그램 계속 실행
```

```
[사용자 프로그램] ─── printf("hello") ───→
[C 라이브러리]   ─── write() 호출 ───→
[시스템 콜 인터페이스] ─── Trap (인터럽트) ───→
[커널]           ─── sys_write() 실행
                 ← 결과 반환 ───
```

### 시스템 콜의 유형

**1. 프로세스 제어**
```c
fork()    // 새 프로세스 생성
exec()    // 프로세스 실행
exit()    // 프로세스 종료
wait()    // 자식 프로세스 대기
getpid()  // 프로세스 ID 반환
```

**2. 파일 관리**
```c
open()    // 파일 열기
close()   // 파일 닫기
read()    // 파일 읽기
write()   // 파일 쓰기
lseek()   // 파일 포인터 이동
```

**3. 장치 관리**
```c
ioctl()   // 장치 제어
read()    // 장치에서 읽기
write()   // 장치에 쓰기
```

**4. 정보 유지**
```c
time()    // 시간 정보
getpid()  // 프로세스 정보
```

**5. 통신**
```c
pipe()    // 파이프 생성
shmget()  // 공유 메모리 생성
socket()  // 소켓 생성
```

### 시스템 콜과 라이브러리 함수의 차이

```c
// 라이브러리 함수: 사용자 모드에서 실행
printf("Hello");  // 내부적으로 write() 시스템 콜 호출

// 시스템 콜: 커널 모드에서 실행
write(1, "Hello", 5);
```

| 구분 | 라이브러리 함수 | 시스템 콜 |
|------|--------------|----------|
| 실행 모드 | 사용자 모드 | 커널 모드 |
| 오버헤드 | 적음 | 모드 전환 비용 |
| 예시 | printf, malloc | write, brk |

### 시스템 콜이 필요한 이유
- 사용자 프로그램이 직접 하드웨어에 접근하면 보안 문제 발생
- 여러 프로그램이 동시에 자원에 접근할 때 충돌 방지
- 하드웨어 추상화를 통한 이식성 확보

### 면접 팁
시스템 콜은 OS의 "문지기" 역할을 합니다. 사용자 프로그램이 아무리 복잡해도 결국 시스템 콜을 통해 OS 서비스를 이용합니다. fork()와 exec()의 차이를 이해하고, 모드 전환의 오버헤드에 대해서도 설명할 수 있으면 좋습니다.

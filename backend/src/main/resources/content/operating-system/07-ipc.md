## 질문
IPC(프로세스 간 통신)란 무엇이며, 어떤 방법들이 있나요?

## 핵심 키워드
- 파이프 (Pipe)
- 메시지 큐 (Message Queue)
- 공유 메모리 (Shared Memory)
- 소켓 (Socket)
- 시그널 (Signal)

## 답변
IPC(Inter-Process Communication)는 프로세스들 사이에 데이터를 주고받는 메커니즘입니다. 프로세스는 독립적인 메모리 공간을 가지므로 직접적인 데이터 공유가 불가능하여 IPC가 필요합니다.

### IPC 방법

**1. 파이프 (Pipe)**
한 방향으로만 데이터가 흐르는 통신 방법입니다.

```c
// 익명 파이프 (부모-자식 프로세스 간)
int fd[2];
pipe(fd);      // fd[0]: 읽기, fd[1]: 쓰기

if (fork() == 0) {  // 자식
    close(fd[1]);
    read(fd[0], buf, sizeof(buf));
} else {             // 부모
    close(fd[0]);
    write(fd[1], "Hello", 5);
}
```

- **익명 파이프**: 부모-자식 관계에서만 사용 가능, 단방향
- **Named Pipe(FIFO)**: 관계없는 프로세스 간에도 사용 가능

**2. 메시지 큐 (Message Queue)**
메시지를 큐에 넣고 받는 방식으로, 커널이 관리합니다.
- 비동기 통신 가능
- 메시지에 타입을 지정하여 선택적 수신 가능
- 파이프보다 유연하지만 커널 메모리 사용

**3. 공유 메모리 (Shared Memory)**
여러 프로세스가 동일한 메모리 영역을 공유합니다.
- 가장 빠른 IPC 방법 (커널 개입 없이 직접 접근)
- 동기화가 필요 (세마포어, 뮤텍스 사용)
- 대용량 데이터 공유에 적합

```c
// 공유 메모리 생성
int shmid = shmget(IPC_PRIVATE, 1024, IPC_CREAT | 0666);
char *data = shmat(shmid, NULL, 0);

// 데이터 쓰기
strcpy(data, "Hello from Process A");

// 다른 프로세스에서 읽기
printf("%s\n", data);  // "Hello from Process A"
```

**4. 소켓 (Socket)**
네트워크를 통한 프로세스 간 통신 방법으로, 다른 시스템의 프로세스와도 통신 가능합니다.
- TCP/UDP 소켓
- Unix Domain Socket (같은 시스템 내)
- 양방향 통신

**5. 시그널 (Signal)**
프로세스에 이벤트를 알리는 소프트웨어 인터럽트입니다.
- SIGKILL, SIGTERM, SIGINT 등
- 간단한 알림에 적합, 데이터 전달에는 부적합

**6. 메모리 맵 파일 (Memory-Mapped File)**
파일을 메모리에 매핑하여 프로세스 간 공유합니다.

### IPC 방법 비교

| 방법 | 속도 | 범위 | 양방향 | 동기화 |
|------|------|------|--------|--------|
| 파이프 | 중간 | 같은 시스템 | 단방향 | 불필요 |
| 메시지 큐 | 중간 | 같은 시스템 | 양방향 | 불필요 |
| 공유 메모리 | 빠름 | 같은 시스템 | 양방향 | 필요 |
| 소켓 | 느림 | 다른 시스템 | 양방향 | 불필요 |
| 시그널 | 빠름 | 같은 시스템 | 단방향 | 불필요 |

### 면접 팁
각 IPC 방법의 장단점을 비교하여 상황에 맞는 방법을 선택할 수 있어야 합니다. 공유 메모리가 가장 빠르지만 동기화 문제가 있고, 소켓은 네트워크를 통해 분산 시스템에서도 사용할 수 있다는 점을 기억하세요.

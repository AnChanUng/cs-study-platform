## 질문
CPU 스케줄링 알고리즘의 종류와 특징에 대해 설명해주세요.

## 핵심 키워드
- 선점형 / 비선점형
- FCFS, SJF, RR, Priority
- 기아(Starvation) 문제
- 멀티레벨 큐
- 응답 시간 / 대기 시간

## 답변
CPU 스케줄링은 Ready Queue에 있는 프로세스 중 어떤 프로세스에 CPU를 할당할지 결정하는 정책입니다. 멀티프로그래밍 환경에서 CPU 이용률을 극대화하기 위해 필수적입니다.

### 선점형 vs 비선점형

**비선점형(Non-preemptive)**: 프로세스가 CPU를 자발적으로 반납할 때까지 빼앗을 수 없음
**선점형(Preemptive)**: 운영체제가 필요에 따라 CPU를 강제로 회수할 수 있음

### 스케줄링 알고리즘

**1. FCFS (First Come First Served) - 비선점**
도착 순서대로 처리합니다.
- 장점: 구현이 간단, 공정
- 단점: 호위 효과(Convoy Effect) - 긴 작업 뒤의 짧은 작업이 오래 대기

```
프로세스: P1(24ms), P2(3ms), P3(3ms)
도착 순서: P1 → P2 → P3
실행: |---P1(24ms)---|P2(3ms)|P3(3ms)|
평균 대기 시간: (0 + 24 + 27) / 3 = 17ms
```

**2. SJF (Shortest Job First) - 비선점/선점(SRTF)**
실행 시간이 가장 짧은 프로세스를 먼저 실행합니다.
- 장점: 평균 대기 시간 최소화 (이론상 최적)
- 단점: 기아(Starvation) 발생 가능, 실행 시간 예측 어려움

**3. Priority Scheduling - 비선점/선점**
우선순위가 높은 프로세스를 먼저 실행합니다.
- 단점: 기아 문제 → **에이징(Aging)** 기법으로 해결 (대기 시간에 따라 우선순위 증가)

**4. Round Robin (RR) - 선점**
각 프로세스에 동일한 시간 할당량(Time Quantum)을 부여하고 순환 실행합니다.

```
프로세스: P1(24ms), P2(3ms), P3(3ms), Time Quantum = 4ms
실행: |P1(4)|P2(3)|P3(3)|P1(4)|P1(4)|P1(4)|P1(4)|P1(4)|
```

- 장점: 응답 시간이 짧음, 공정
- 단점: Time Quantum 크기에 따라 성능이 달라짐
  - 너무 크면 FCFS와 같아짐
  - 너무 작으면 Context Switching 오버헤드 증가

**5. 멀티레벨 큐 (Multilevel Queue)**
프로세스를 여러 그룹으로 분류하고 각 그룹별 다른 스케줄링 적용합니다.

```
높은 우선순위 ← [시스템 프로세스]     (RR)
              [대화형 프로세스]      (RR)
              [배치 프로세스]        (FCFS)
낮은 우선순위 ← [학생 프로세스]      (FCFS)
```

**6. 멀티레벨 피드백 큐 (MLFQ)**
프로세스가 큐 사이를 이동할 수 있는 유연한 스케줄링입니다. 가장 일반적으로 사용됩니다.

### 평가 기준
- **CPU 이용률**: CPU가 작업하는 시간의 비율
- **처리량(Throughput)**: 단위 시간당 완료된 프로세스 수
- **대기 시간**: Ready Queue에서 대기한 시간
- **응답 시간**: 요청부터 첫 응답까지의 시간
- **반환 시간(Turnaround Time)**: 제출부터 완료까지의 시간

### 면접 팁
실제 OS에서는 단일 알고리즘보다 MLFQ 같은 복합적인 방식을 사용합니다. 각 알고리즘의 장단점과 기아 문제 해결 방법(에이징)을 함께 설명하세요.

3. 실시간 시스템에서는 어떤 스케줄링 알고리즘을 사용하나요?

## 질문
페이지 교체 알고리즘의 종류와 각각의 특징에 대해 설명해주세요.

## 핵심 키워드
- 페이지 폴트 (Page Fault)
- FIFO, LRU, LFU, OPT
- 벨라디의 모순 (Belady's Anomaly)
- 지역성 원리
- 스래싱 (Thrashing)

## 답변
페이지 교체 알고리즘은 물리 메모리가 부족할 때 어떤 페이지를 교체할지 결정하는 알고리즘입니다. 페이지 폴트(Page Fault)가 발생하고 빈 프레임이 없을 때 사용됩니다.

### 페이지 폴트 처리 과정
```
1. CPU가 페이지 참조 → 페이지 테이블 확인
2. 해당 페이지가 메모리에 없음 → 페이지 폴트 발생
3. 디스크에서 해당 페이지를 가져옴
4. 빈 프레임이 없으면 → 페이지 교체 알고리즘으로 희생 페이지 선택
5. 희생 페이지를 디스크에 쓰고 새 페이지를 적재
6. 페이지 테이블 갱신
7. 프로세스 재실행
```

### 알고리즘 종류

**1. OPT (Optimal)**
가장 오랫동안 사용되지 않을 페이지를 교체합니다.
- 이론적으로 최적이지만 미래를 예측할 수 없어 구현 불가
- 다른 알고리즘의 성능 비교 기준으로 사용

```
참조 문자열: 7 0 1 2 0 3 0 4 2 3 (프레임 수: 3)
OPT: 페이지 폴트 6회
```

**2. FIFO (First In First Out)**
가장 먼저 들어온 페이지를 교체합니다.
- 구현이 간단
- **벨라디의 모순**: 프레임 수를 늘려도 페이지 폴트가 증가할 수 있음

```
참조 문자열: 1 2 3 4 1 2 5 1 2 3 4 5
프레임 3개: 9번 페이지 폴트
프레임 4개: 10번 페이지 폴트 (오히려 증가!)
```

**3. LRU (Least Recently Used)**
가장 오랫동안 사용되지 않은 페이지를 교체합니다.
- 지역성 원리에 기반한 가장 널리 사용되는 알고리즘
- 벨라디의 모순이 발생하지 않음
- 구현 비용이 높음 (참조 시간을 기록해야 함)

```
참조 문자열: 7 0 1 2 0 3 0 4 2 3 (프레임 수: 3)

Step 1: [7]       ← 7 적재
Step 2: [7,0]     ← 0 적재
Step 3: [7,0,1]   ← 1 적재
Step 4: [2,0,1]   ← 7 교체 (가장 오래전에 사용)
Step 5: [2,0,1]   ← 0 히트
Step 6: [2,0,3]   ← 1 교체 (가장 오래전에 사용)
```

**4. LFU (Least Frequently Used)**
참조 횟수가 가장 적은 페이지를 교체합니다.
- 초기에 많이 사용되었지만 이후 사용되지 않는 페이지가 남는 문제
- 참조 횟수가 같으면 LRU 방식으로 결정

**5. Clock Algorithm (Second Chance)**
FIFO의 개선 버전으로, 참조 비트를 확인합니다.
- 원형 큐에서 참조 비트가 0인 페이지를 교체
- 참조 비트가 1이면 0으로 변경하고 다음 페이지 확인
- LRU의 근사 알고리즘으로 실제 OS에서 많이 사용

### 스래싱 (Thrashing)
프로세스의 실행 시간보다 페이지 교체에 더 많은 시간을 소비하는 현상입니다. 멀티프로그래밍 정도가 과도하면 발생합니다.

**해결 방법:**
- Working Set 모델: 프로세스가 일정 기간 동안 참조하는 페이지 집합을 메모리에 유지
- Page Fault Frequency: 페이지 폴트율을 기반으로 프레임 수 조절

### 면접 팁
LRU가 가장 자주 출제됩니다. 구현 방법(HashMap + DoublyLinkedList)을 코드로 작성할 수 있어야 합니다. 벨라디의 모순이 FIFO에서만 발생하고 LRU에서는 발생하지 않는 이유도 설명할 수 있으면 좋습니다.

## 질문
버블 정렬(Bubble Sort)의 동작 원리와 시간 복잡도에 대해 설명해주세요.

## 핵심 키워드
- 인접 요소 비교/교환
- O(n^2) 시간 복잡도
- 안정 정렬
- 제자리 정렬
- 최적화 (early termination)

## 답변
버블 정렬은 인접한 두 요소를 비교하여 순서가 맞지 않으면 교환하는 과정을 반복하는 정렬 알고리즘입니다. 가장 큰 요소가 거품처럼 배열의 끝으로 이동하는 모습에서 이름이 유래했습니다.

### 동작 과정

```
초기: [5, 3, 8, 4, 2]

1회전: 가장 큰 값 8이 끝으로 이동
[3, 5, 8, 4, 2] → [3, 5, 4, 8, 2] → [3, 5, 4, 2, 8]

2회전: 5가 제자리로
[3, 5, 4, 2, 8] → [3, 4, 5, 2, 8] → [3, 4, 2, 5, 8]

3회전: 4가 제자리로
[3, 4, 2, 5, 8] → [3, 2, 4, 5, 8]

4회전: 3이 제자리로
[2, 3, 4, 5, 8] → 정렬 완료!
```

### 구현

```java
void bubbleSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        boolean swapped = false; // 최적화: 교환 여부 체크
        for (int j = 0; j < n - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                // 인접 요소 교환
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
                swapped = true;
            }
        }
        if (!swapped) break; // 교환이 없으면 이미 정렬된 상태
    }
}
```

### 시간 복잡도

| 경우 | 시간 복잡도 | 설명 |
|------|-----------|------|
| 최선 | O(n) | 이미 정렬된 경우 (최적화 적용 시) |
| 평균 | O(n^2) | |
| 최악 | O(n^2) | 역순 정렬된 경우 |

- 공간 복잡도: O(1) (제자리 정렬)
- 안정 정렬: 동일한 값의 상대적 순서가 유지됨

### 장단점
**장점:** 구현이 매우 간단, 안정 정렬, 추가 메모리 불필요
**단점:** O(n^2)으로 비효율적, 실무에서 거의 사용되지 않음

### 면접 팁
버블 정렬은 교육적 목적으로 중요하지만 실무에서는 사용하지 않습니다. 최적화(early termination)를 적용하면 이미 정렬된 배열에 대해 O(n)이 된다는 점을 언급하세요.

## 질문
선택 정렬(Selection Sort)의 동작 원리와 특징에 대해 설명해주세요.

## 핵심 키워드
- 최소값 선택
- O(n^2) 시간 복잡도
- 불안정 정렬
- 제자리 정렬
- 교환 횟수 최소

## 답변
선택 정렬은 배열에서 가장 작은(또는 큰) 요소를 선택하여 정렬되지 않은 부분의 첫 번째 위치에 놓는 과정을 반복하는 알고리즘입니다.

### 동작 과정

```
초기: [64, 25, 12, 22, 11]

1회전: 최소값 11 선택 → 첫 번째와 교환
[11, 25, 12, 22, 64]

2회전: 나머지에서 최소값 12 선택
[11, 12, 25, 22, 64]

3회전: 나머지에서 최소값 22 선택
[11, 12, 22, 25, 64]

4회전: 나머지에서 최소값 25 선택
[11, 12, 22, 25, 64] → 정렬 완료
```

### 구현

```java
void selectionSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        int minIdx = i;
        // 최소값의 인덱스 찾기
        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[minIdx]) {
                minIdx = j;
            }
        }
        // 교환
        int temp = arr[i];
        arr[i] = arr[minIdx];
        arr[minIdx] = temp;
    }
}
```

### 시간 복잡도

| 경우 | 시간 복잡도 |
|------|-----------|
| 최선 | O(n^2) |
| 평균 | O(n^2) |
| 최악 | O(n^2) |

- 공간 복잡도: O(1)
- **불안정 정렬**: 동일한 값의 순서가 바뀔 수 있음
- 비교 횟수: n(n-1)/2 (항상 동일)
- 교환 횟수: O(n) (버블 정렬보다 적음)

### 장단점
**장점:** 구현이 단순, 교환 횟수가 적어 교환 비용이 클 때 유리
**단점:** 항상 O(n^2), 이미 정렬되어 있어도 동일한 비교 수행

### 면접 팁
선택 정렬의 핵심 특징은 교환 횟수가 O(n)으로 최소라는 점입니다. 메모리 쓰기가 비용이 큰 환경에서는 버블 정렬보다 유리할 수 있습니다. 하지만 어떤 경우에도 O(n^2)이라는 한계가 있습니다.

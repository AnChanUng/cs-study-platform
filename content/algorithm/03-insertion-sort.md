## 질문
삽입 정렬(Insertion Sort)의 동작 원리와 특징에 대해 설명해주세요.

## 핵심 키워드
- 정렬된 부분에 삽입
- O(n^2) 시간 복잡도
- 안정 정렬
- 거의 정렬된 데이터에 효율적
- 온라인 알고리즘

## 답변
삽입 정렬은 배열의 각 요소를 이미 정렬된 부분의 적절한 위치에 삽입하는 방식으로 동작합니다. 카드 게임에서 손에 들고 있는 카드를 정렬하는 방식과 유사합니다.

### 동작 과정

```
초기: [5, 3, 4, 1, 2]

1회전: 3을 삽입 → [3, 5, 4, 1, 2]
2회전: 4를 삽입 → [3, 4, 5, 1, 2]
3회전: 1을 삽입 → [1, 3, 4, 5, 2]
4회전: 2를 삽입 → [1, 2, 3, 4, 5] 완료!
```

### 구현

```java
void insertionSort(int[] arr) {
    for (int i = 1; i < arr.length; i++) {
        int key = arr[i];
        int j = i - 1;

        // key보다 큰 요소들을 오른쪽으로 이동
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;
    }
}
```

### 시간 복잡도

| 경우 | 시간 복잡도 | 설명 |
|------|-----------|------|
| 최선 | O(n) | 이미 정렬된 경우 |
| 평균 | O(n^2) | |
| 최악 | O(n^2) | 역순 정렬된 경우 |

- 공간 복잡도: O(1)
- 안정 정렬

### 장단점
**장점:**
- 거의 정렬된 데이터에서 O(n)에 가까운 성능
- 구현이 간단하고 안정 정렬
- 데이터가 추가될 때마다 정렬 가능 (온라인 알고리즘)
- 작은 데이터셋에서 효율적

**단점:**
- 평균/최악 O(n^2)

### 실무 활용
- Tim Sort(Java, Python의 기본 정렬)는 내부적으로 작은 부분 배열에 삽입 정렬을 사용합니다
- 데이터가 거의 정렬되어 있을 때 퀵 정렬보다 빠를 수 있습니다

### 면접 팁
삽입 정렬은 O(n^2) 정렬 알고리즘 중 실무에서 가장 많이 활용됩니다. Tim Sort가 삽입 정렬을 사용하는 이유를 설명할 수 있으면 좋습니다. 또한 이진 삽입 정렬(Binary Insertion Sort)도 언급할 수 있습니다.

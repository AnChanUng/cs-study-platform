## 질문
배열(Array)의 특징과 시간 복잡도에 대해 설명해주세요.

## 핵심 키워드
- 연속적인 메모리 공간
- 인덱스를 통한 랜덤 접근
- 고정 크기
- O(1) 접근
- 캐시 지역성

## 답변
배열(Array)은 동일한 타입의 데이터를 연속적인 메모리 공간에 저장하는 가장 기본적인 자료구조입니다. 인덱스를 사용하여 요소에 빠르게 접근할 수 있습니다.

### 특징

**1. 연속적인 메모리 할당**
배열의 요소들은 메모리에서 연속적으로 배치됩니다. 시작 주소와 인덱스로 요소의 주소를 계산할 수 있습니다.
```
요소 주소 = 시작 주소 + (인덱스 × 요소 크기)
```

**2. 고정 크기**
배열은 생성 시 크기가 결정되며, 이후 변경할 수 없습니다. 크기 변경이 필요하면 새로운 배열을 생성하고 데이터를 복사해야 합니다.

```java
int[] arr = new int[5]; // 크기 5로 고정
arr[0] = 10;
arr[1] = 20;

// 접근: O(1)
System.out.println(arr[2]); // 인덱스로 바로 접근
```

### 시간 복잡도

| 연산 | 시간 복잡도 | 설명 |
|------|-----------|------|
| 접근 (Access) | O(1) | 인덱스로 직접 접근 |
| 검색 (Search) | O(n) | 순차 탐색 필요 |
| 삽입 (Insert) | O(n) | 요소 이동 필요 |
| 삭제 (Delete) | O(n) | 요소 이동 필요 |
| 끝에 추가 | O(1) | 공간이 있는 경우 |

### 장점
- **빠른 접근 속도**: 인덱스를 통한 O(1) 랜덤 접근
- **캐시 지역성(Cache Locality)**: 연속 메모리로 CPU 캐시 효율이 좋음
- **메모리 효율**: 포인터 등 추가 메모리가 불필요

### 단점
- **고정 크기**: 크기 변경이 불가능하여 메모리 낭비 또는 부족 발생 가능
- **삽입/삭제 비효율**: 중간에 삽입/삭제 시 요소 이동 필요 (O(n))
- **메모리 단편화**: 큰 연속 메모리 공간이 필요

### 배열이 적합한 경우
- 데이터의 크기가 미리 정해져 있을 때
- 인덱스를 통한 빠른 접근이 필요할 때
- 삽입/삭제보다 조회가 빈번할 때

### 면접 팁
배열의 핵심은 "연속 메모리"와 "O(1) 랜덤 접근"입니다. LinkedList와의 비교를 통해 각각의 장단점을 명확히 설명할 수 있어야 합니다. 캐시 지역성 개념도 언급하면 좋습니다.

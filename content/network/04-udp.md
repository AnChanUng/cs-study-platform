## 질문
UDP란 무엇이며, TCP와의 차이점에 대해 설명해주세요.

## 핵심 키워드
- 비연결형 프로토콜
- 신뢰성 없는 전송
- 데이터그램
- 낮은 오버헤드
- 실시간 통신

## 답변
UDP(User Datagram Protocol)는 전송 계층의 비연결형 프로토콜로, 별도의 연결 설정 없이 데이터를 전송합니다. 신뢰성보다 속도가 중요한 애플리케이션에서 사용합니다.

### UDP 헤더 구조

```
 0      8      16     24     32 bits
├──────┼──────┼──────┼──────┤
│  출발지 포트  │  목적지 포트  │
├──────────────┼──────────────┤
│   전체 길이   │   체크섬     │
├──────────────┴──────────────┤
│          데이터              │
└─────────────────────────────┘
(헤더 크기: 8바이트 - TCP의 20바이트보다 훨씬 작음)
```

### UDP 특징

**1. 비연결형**: 3-way handshake 없이 바로 데이터 전송
**2. 비신뢰성**: 순서 보장 없음, 손실 감지/재전송 없음
**3. 낮은 오버헤드**: 헤더가 작고 처리가 간단
**4. 빠른 전송**: 연결 설정/해제 과정 없음
**5. 브로드캐스트/멀티캐스트 지원**

### TCP vs UDP 비교

| 특성 | TCP | UDP |
|------|-----|-----|
| 연결 방식 | 연결형 (Connection-oriented) | 비연결형 (Connectionless) |
| 신뢰성 | 보장 (ACK, 재전송) | 보장하지 않음 |
| 순서 보장 | 보장 | 보장하지 않음 |
| 흐름/혼잡 제어 | 있음 | 없음 |
| 전송 속도 | 상대적으로 느림 | 빠름 |
| 헤더 크기 | 20~60바이트 | 8바이트 |
| 통신 방식 | 1:1 | 1:1, 1:N, N:N |

### UDP 활용 사례

**1. 실시간 스트리밍**
- 동영상, 음악 스트리밍
- 약간의 패킷 손실은 허용, 지연이 더 중요

**2. 온라인 게임**
- 캐릭터 위치 등 빈번한 업데이트
- 오래된 데이터를 재전송하는 것보다 새 데이터가 중요

**3. DNS (Domain Name System)**
- 작은 크기의 질의/응답
- 빠른 응답이 중요

**4. VoIP (인터넷 전화)**
- 실시간 음성 전달
- 지연보다 약간의 손실이 나음

**5. IoT, 센서 데이터**
- 대량의 작은 데이터를 빠르게 전송
- 일부 손실 허용

### UDP 위에서 신뢰성 구현

UDP 자체는 신뢰성이 없지만, 애플리케이션 레벨에서 구현할 수 있습니다.
- **QUIC**: Google이 개발한 UDP 기반 프로토콜 (HTTP/3에서 사용)
- **RTP**: 실시간 미디어 전송 프로토콜
- **게임 프로토콜**: 자체적인 ACK, 재전송 로직 구현

### 면접 팁
"UDP는 신뢰성이 없어서 나쁘다"가 아니라, 상황에 따라 TCP보다 적합할 수 있다는 점을 강조하세요. HTTP/3이 UDP 기반의 QUIC를 사용한다는 최신 트렌드도 알아두면 좋습니다.

## 질문
인터럽트란 무엇이며, 처리 과정에 대해 설명해주세요.

## 핵심 키워드
- 하드웨어 인터럽트
- 소프트웨어 인터럽트 (Trap)
- 인터럽트 서비스 루틴 (ISR)
- 인터럽트 벡터 테이블
- 인터럽트 우선순위

## 답변
인터럽트(Interrupt)는 CPU가 현재 실행 중인 작업을 일시 중단하고 우선적으로 처리해야 할 이벤트가 발생했을 때, 해당 이벤트를 처리한 후 다시 원래 작업으로 복귀하는 메커니즘입니다.

### 인터럽트의 종류

**1. 하드웨어 인터럽트**
외부 하드웨어 장치에 의해 발생합니다.
- 키보드 입력, 마우스 클릭
- 디스크 I/O 완료
- 타이머 인터럽트 (프로세스 스케줄링에 사용)
- 네트워크 패킷 도착

**2. 소프트웨어 인터럽트 (Trap)**
소프트웨어적으로 발생하는 인터럽트입니다.
- **시스템 콜**: 프로그램이 OS 서비스를 요청
- **예외(Exception)**: 0으로 나누기, 잘못된 메모리 접근
- **디버거 브레이크포인트**

### 인터럽트 처리 과정

```
1. 인터럽트 발생
2. CPU: 현재 실행 중인 명령어 완료
3. CPU: 현재 상태(PC, 레지스터 등)를 스택에 저장
4. 인터럽트 벡터 테이블에서 해당 ISR의 주소를 찾음
5. ISR(Interrupt Service Routine) 실행
6. 이전 상태를 복원하고 원래 작업으로 복귀
```

```
인터럽트 벡터 테이블:
┌─────────┬──────────────────┐
│ 번호    │ ISR 주소          │
├─────────┼──────────────────┤
│ 0       │ 0x0000 (0 나누기) │
│ 1       │ 0x0100 (키보드)   │
│ 2       │ 0x0200 (타이머)   │
│ ...     │ ...              │
└─────────┴──────────────────┘
```

### 인터럽트와 폴링 비교

**인터럽트 방식:**
- 이벤트 발생 시 CPU에 알림
- CPU가 다른 작업을 수행할 수 있음
- 효율적이고 반응이 빠름

**폴링(Polling) 방식:**
- CPU가 주기적으로 상태를 확인
- CPU 자원을 낭비할 수 있음
- 구현이 간단

### 인터럽트 우선순위
여러 인터럽트가 동시에 발생할 경우 우선순위에 따라 처리합니다.
```
전원 이상 > 기계 장애 > 외부 인터럽트 > I/O > 프로그램 검사 > SVC
```

인터럽트 처리 중에 더 높은 우선순위의 인터럽트가 발생하면 현재 ISR을 중단하고 새로운 인터럽트를 먼저 처리합니다 (중첩 인터럽트).

### 면접 팁
인터럽트는 OS의 핵심 동작 원리입니다. 특히 시스템 콜이 소프트웨어 인터럽트(Trap)를 통해 구현된다는 점을 연결하여 설명하면 좋습니다. 인터럽트 없이는 멀티태스킹이 불가능합니다.

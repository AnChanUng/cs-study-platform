## 질문
데드락(교착 상태)이란 무엇이며, 발생 조건과 해결 방법에 대해 설명해주세요.

## 핵심 키워드
- 상호 배제 (Mutual Exclusion)
- 점유 대기 (Hold and Wait)
- 비선점 (No Preemption)
- 환형 대기 (Circular Wait)
- 데드락 예방/회피/탐지/회복

## 답변
데드락(Deadlock)은 두 개 이상의 프로세스가 서로가 가진 자원을 기다리면서 무한히 대기하는 상태입니다.

### 데드락 예시
```
프로세스 A: 자원 1을 보유, 자원 2를 대기
프로세스 B: 자원 2를 보유, 자원 1을 대기

→ 서로 상대방의 자원을 기다리며 무한 대기
```

```java
// 데드락 발생 예시
Thread t1 = new Thread(() -> {
    synchronized (lockA) {          // lockA 획득
        Thread.sleep(100);
        synchronized (lockB) {      // lockB 대기 → 데드락!
            // ...
        }
    }
});

Thread t2 = new Thread(() -> {
    synchronized (lockB) {          // lockB 획득
        Thread.sleep(100);
        synchronized (lockA) {      // lockA 대기 → 데드락!
            // ...
        }
    }
});
```

### 데드락 발생 조건 (4가지 모두 만족해야 발생)

1. **상호 배제 (Mutual Exclusion)**: 자원은 한 번에 하나의 프로세스만 사용 가능
2. **점유 대기 (Hold and Wait)**: 자원을 보유한 상태에서 다른 자원을 대기
3. **비선점 (No Preemption)**: 다른 프로세스가 보유한 자원을 강제로 빼앗을 수 없음
4. **환형 대기 (Circular Wait)**: 프로세스 간 자원 대기 관계가 원형을 이룸

### 데드락 해결 방법

**1. 예방 (Prevention)**
4가지 조건 중 하나 이상을 원천적으로 차단합니다.
- 점유 대기 제거: 모든 자원을 한꺼번에 요청
- 비선점 제거: 자원을 강제로 회수 가능하게 함
- 환형 대기 제거: 자원에 순서를 부여하여 순서대로만 요청

```java
// 환형 대기 제거: 항상 lockA → lockB 순서로 획득
synchronized (lockA) {
    synchronized (lockB) {
        // 안전
    }
}
```

**2. 회피 (Avoidance)**
시스템이 안전한 상태(Safe State)를 유지하도록 자원 할당을 결정합니다.
- **은행원 알고리즘(Banker's Algorithm)**: 자원 요청 시 안전 상태를 유지할 수 있으면 할당, 아니면 대기

**3. 탐지 (Detection)**
데드락 발생을 허용하고, 주기적으로 탐지합니다.
- 자원 할당 그래프에서 사이클 탐지
- 탐지 알고리즘을 주기적으로 실행

**4. 회복 (Recovery)**
데드락이 탐지되면 해결합니다.
- 프로세스 종료 (일부 또는 전체)
- 자원 강제 선점 (일부 프로세스의 자원을 회수)
- 롤백 (체크포인트로 되돌림)

### 면접 팁
데드락의 4가지 조건을 정확히 기억하고, 각 조건을 제거하여 예방하는 방법을 설명하세요. 실무에서는 주로 데이터베이스의 Lock 경합으로 데드락이 발생하며, DB는 보통 탐지 후 하나의 트랜잭션을 롤백하는 방식으로 해결합니다.

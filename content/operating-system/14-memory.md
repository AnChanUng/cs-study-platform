## 질문
운영체제의 메모리 관리 기법에 대해 설명해주세요.

## 핵심 키워드
- 연속/비연속 메모리 할당
- 내부/외부 단편화
- 가상 메모리
- 스와핑 (Swapping)
- 요구 페이징 (Demand Paging)

## 답변
메모리 관리는 운영체제의 핵심 기능 중 하나로, 한정된 물리 메모리를 효율적으로 사용하고 프로세스 간 메모리 보호를 제공하는 것을 목표로 합니다.

### 메모리 할당 방식

**1. 연속 메모리 할당**
각 프로세스를 메모리의 연속된 공간에 배치합니다.

- **고정 분할 방식**: 메모리를 고정 크기로 분할. 내부 단편화 발생
- **가변 분할 방식**: 프로세스 크기에 맞게 분할. 외부 단편화 발생

가변 분할의 배치 전략:
- **First Fit**: 첫 번째로 충분한 공간을 찾아 배치
- **Best Fit**: 가장 작은 충분한 공간에 배치
- **Worst Fit**: 가장 큰 공간에 배치

**2. 비연속 메모리 할당**
프로세스를 여러 조각으로 나누어 비연속적으로 배치합니다.
- 페이징 (Paging): 고정 크기 분할
- 세그먼테이션 (Segmentation): 가변 크기 분할

### 단편화 (Fragmentation)

**내부 단편화 (Internal Fragmentation)**
할당된 메모리 공간 중 사용되지 않는 부분입니다. 고정 크기 분할에서 발생합니다.
```
할당 블록: 4KB, 실제 사용: 3KB → 1KB 낭비
```

**외부 단편화 (External Fragmentation)**
총 빈 공간은 충분하지만 연속적이지 않아 할당할 수 없는 상태입니다.
```
[사용중][빈3KB][사용중][빈2KB][사용중][빈4KB]
→ 총 9KB 비었지만 7KB 프로세스 배치 불가
```

해결: 압축(Compaction) - 사용 중인 메모리를 한쪽으로 몰아 큰 빈 공간을 만듦

### 가상 메모리 (Virtual Memory)

물리 메모리보다 큰 주소 공간을 프로세스에 제공하는 기법입니다. 실제로 사용하는 부분만 물리 메모리에 적재하고, 나머지는 디스크(스왑 영역)에 저장합니다.

**요구 페이징 (Demand Paging)**
- 필요한 페이지만 메모리에 적재하는 방식
- 처음에는 아무 페이지도 적재하지 않고, 접근 시 페이지 폴트가 발생하면 적재
- 지역성 원리(Locality of Reference)에 의해 효율적으로 동작

```
지역성 원리:
- 시간적 지역성: 최근 접근한 데이터에 다시 접근할 가능성이 높음
- 공간적 지역성: 인접한 데이터에 접근할 가능성이 높음
```

### 스와핑 (Swapping)
프로세스 전체를 디스크로 내보내고(swap-out), 필요할 때 다시 메모리로 불러오는(swap-in) 기법입니다. 메모리가 부족할 때 사용합니다.

### 면접 팁
가상 메모리가 필요한 이유(물리 메모리의 한계, 프로세스 보호)와 요구 페이징의 동작 원리를 잘 설명하세요. 페이지 폴트 발생 시의 처리 과정도 중요합니다.

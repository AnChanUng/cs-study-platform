## 답변
파일 시스템(File System)은 운영체제가 디스크에 데이터를 저장하고 관리하는 체계입니다. 사용자가 파일 이름으로 데이터에 접근할 수 있게 하며, 파일의 생성, 삭제, 읽기, 쓰기를 관리합니다.

### 파일 시스템 구성요소

**파일(File)**: 관련된 데이터의 논리적 집합
**디렉토리(Directory)**: 파일 정보를 포함하는 특수한 파일
**메타데이터**: 파일 이름, 크기, 생성일, 권한, 위치 정보 등

### 디스크 할당 방식

**1. 연속 할당 (Contiguous Allocation)**
파일을 연속된 블록에 저장합니다.
- 장점: 순차/직접 접근 모두 빠름
- 단점: 외부 단편화 발생, 파일 크기 변경이 어려움

**2. 연결 할당 (Linked Allocation)**
각 블록이 다음 블록을 가리키는 포인터를 포함합니다.
- 장점: 외부 단편화 없음, 파일 크기 유연
- 단점: 직접 접근 불가(순차 접근만), 포인터 손상 시 데이터 손실
- FAT 파일 시스템이 이 방식을 개선하여 사용

**3. 인덱스 할당 (Indexed Allocation)**
인덱스 블록에 모든 데이터 블록의 포인터를 저장합니다.
- 장점: 직접 접근 가능, 외부 단편화 없음
- 단점: 인덱스 블록을 위한 추가 공간 필요
- Unix/Linux의 inode 방식이 이를 사용

### inode (Index Node) 구조

Unix/Linux 파일 시스템에서 파일의 메타데이터를 저장하는 자료구조입니다.

```
inode:
├── 파일 타입, 권한
├── 소유자, 그룹
├── 파일 크기
├── 타임스탬프 (생성, 수정, 접근)
├── 직접 블록 포인터 (12개)       → 데이터 블록
├── 단일 간접 포인터 (1개)         → 포인터 블록 → 데이터 블록
├── 이중 간접 포인터 (1개)         → 포인터 블록 → 포인터 블록 → 데이터
└── 삼중 간접 포인터 (1개)         → ...
```

### 주요 파일 시스템

| 파일 시스템 | OS | 특징 |
|------------|-----|------|
| FAT32 | Windows | 단순, 호환성 높음, 4GB 파일 제한 |
| NTFS | Windows | 저널링, 대용량, 보안 기능 |
| ext4 | Linux | 저널링, 대용량, inode 기반 |
| APFS | macOS | 스냅샷, 암호화, SSD 최적화 |

### 저널링 (Journaling)
파일 시스템의 변경 사항을 로그(저널)에 먼저 기록한 후 실제 작업을 수행합니다. 시스템 장애 발생 시 저널을 이용하여 일관된 상태로 복구할 수 있습니다.

```
1. 변경 사항을 저널에 기록 (Write-Ahead Logging)
2. 실제 데이터 영역에 반영
3. 저널에서 해당 기록 삭제
→ 중간에 장애가 발생해도 저널을 보고 복구 가능
```

### 면접 팁
파일 시스템의 기본 구조와 inode의 개념을 이해하세요. 실무에서는 파일 시스템 선택보다는 디스크 I/O 최적화, 캐싱 전략이 중요합니다. 저널링이 데이터 안정성을 위해 어떤 역할을 하는지 설명할 수 있으면 좋습니다.

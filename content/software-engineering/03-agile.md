## 질문
애자일(Agile) 방법론이란 무엇이며, 전통적인 워터폴 방법론과의 차이점을 설명해주세요.

## 핵심 키워드
- 애자일 선언문
- 반복적 개발 (Iterative)
- 스크럼 (Scrum)
- 칸반 (Kanban)
- 스프린트

## 답변
애자일(Agile)은 소프트웨어 개발에서 변화에 빠르게 대응하고, 고객과의 지속적인 소통을 통해 점진적으로 소프트웨어를 개발하는 방법론입니다.

### 애자일 선언문 (4가지 가치)

1. 프로세스와 도구보다 **개인과 상호작용**을
2. 포괄적인 문서보다 **작동하는 소프트웨어**를
3. 계약 협상보다 **고객과의 협력**을
4. 계획을 따르기보다 **변화에 대응**하기를

### 워터폴 vs 애자일

| 구분 | 워터폴 | 애자일 |
|------|--------|--------|
| 개발 방식 | 순차적 | 반복적, 점진적 |
| 요구사항 | 초기 확정 | 지속적 변경 가능 |
| 고객 참여 | 초기/최종 | 전 과정 |
| 산출물 | 각 단계별 문서 | 작동하는 소프트웨어 |
| 변경 대응 | 어려움 | 유연 |
| 릴리즈 | 전체 완료 후 | 주기적 (2~4주) |

### 스크럼 (Scrum)

가장 널리 사용되는 애자일 프레임워크입니다.

**역할:**
- **Product Owner**: 제품 백로그 관리, 우선순위 결정
- **Scrum Master**: 프로세스 지원, 장애물 제거
- **Development Team**: 개발 수행 (5~9명)

**이벤트:**
- **스프린트 (Sprint)**: 1~4주의 개발 주기
- **스프린트 플래닝**: 스프린트 목표와 작업 계획
- **데일리 스크럼**: 매일 15분 회의 (어제/오늘/장애물)
- **스프린트 리뷰**: 결과물 시연
- **스프린트 회고**: 프로세스 개선점 논의

**산출물:**
- 제품 백로그 (Product Backlog)
- 스프린트 백로그 (Sprint Backlog)
- 인크리먼트 (Increment)

### 칸반 (Kanban)

시각적 보드를 통해 작업 흐름을 관리합니다.
```
| To Do | In Progress | Review | Done |
|-------|-------------|--------|------|
| 기능A  | 기능B       | 기능C   | 기능D |
| 기능E  |             |        |      |

WIP(Work In Progress) 제한으로 과부하 방지
```

### 면접 팁
실무에서 애자일을 적용한 경험을 구체적으로 설명할 수 있으면 좋습니다. 애자일은 "프레임워크"가 아니라 "마인드셋"이라는 점을 강조하세요. 또한 애자일의 한계(대규모 프로젝트, 고정 요구사항)도 함께 언급하면 균형 잡힌 답변이 됩니다.
